edas学习

edas 容错

hsf 本身不提供重连的能力

dobble 本身提供配置，在框架中就会充重连

区别：商业化是hsf 开源是dobble

幂等性（数据一致性）使用业务id，去重，避免消费同一条信息

服务提供者 -> 配置中心 -> 服务调用者

限流：为了服务安全，edas可以直接限制某个接口的请求登记

降级： 服务提供者作为服务调用者，另外一个服务级别不够，该服务自动降级到另外一个服务的级别

traceId 每个调用链的唯一id

会话Id

采样率： 默认的10% ，日志量信息量控制

大数据集群 : jstrom

edas： 回滚

应用健康检查：提供一个自动化脚本，自动检测核心功能，这个地址会定期检查

jvm参数配置，理论预留总内存的1/4

日志： tomcat日志

​ edas 注册服务的日志

​ hsf 日志

cpu rt load指标

超时可能引起的原因：

  1. 服务端业务逻辑慢，时间超过

  2. 网络问题

  3. 服务器端或者调用段gc
  4. 服务端或者调用端 load, cpu过高
  5. 传输大对象导致序列化慢
  6. 服务设置时间过短

jvm 性能优化，参数配置

arms 应用监控
