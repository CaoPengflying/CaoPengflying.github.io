mapreduce优化

发表于 2018-05-10 | 分类于 hadoop | 阅读次数:

hadoop处理数据的模块是mapreduce模块
mapreduce模块处理数据分析总共分为三个部分

  1. map
    mapreduce是处理hadoop上的hdfs中的文件，map阶段可以将文件进行读取
    按行读取，他的输入就是hdfs文件，输出的是以下格式
    <0, java hadoop linux ….>
    读取成一行的内容，然后要对数据进行处理
    生成类似的格式< java, 1>< hadoop, 1> …
    hadoop处理大数据的思路就是将数据进行分区，一行一行的去读取，去处理
    处理之后将数据分发给shuffle

  2. shuffle
    shuffle接收到了map传过来的数据，他会对数据进行分区，分区的目的是可以让多台机器同时对数据进行处理，但是同一个内容的处理不能分给不同的机器，所以在该阶段进行分区，确保数据的处理，同一数据要在一台机器上运算。
    然后对数据进行排序。
    shuffle会将map传过来的数据进行处理，这个处理也要有io，他会分配一定大小的内存，100m 当超过这个80%时，会采用小文件的方式去处理，将数据全部转移到小文件中，然后最终再合并到一起，然后再重新排序。然后将结果分发给reduce
    数据格式：< had, <1,1,1,1,1,1>>

  1. reduce
    reduce接收到shuffle的数据后，对数据进行处理，得到最终的结果

整个阶段如果想要优化，可以从shuffle和reduce两个阶段进行，map阶段一般没有什么优化
shuffle优化：

  1. 可以将reduce阶段的任务，合并到shuffle中来，当然这个阶段前提是符合加法分配律的可以采用该方式

  2. 在shuffle阶段有文件的合并，所以可以将文件进行压缩。

压缩的属性可以在hadoop官方文档去查看，设置一下分别是两个文件的压缩。可以以进行优化
