Junit单元测试-TDD

单元测试是什么?

单元测试，有单元和测试两个词组成。单元就是最小单位，测试就验证正确性。合起来就是针对程序的最小单位进行正确性检验的测试工作。

为什么要单元测试?

  1. 减少bug
  2. 快速定位bug
  3. 提高代码质量
  4. 减少调试时间

…….

一个机器，由各种细小的零件组成，如果其中某件零件坏了，机器运行故障。必须保证每个零件都按设计图要求的规格，机器才能正常运行。各种零件都是可拆卸并且可维修。

在实际开发当中，很多开发人员在开发灵感强的时候会从头一直写到最下面，都在一个方法里面，这样就不是各种零件可拆卸并且可维修了，每次有bug，都要把整个程序都要检查一遍，这样的成本就很高。单元测试的意义有好多，从结果上看就是为了写出清洁的代码(高质量，bug少，阅读性高)，也就是说在写代码过程中，只要是方便单元测试的，都是清洁的代码。

那些人在写单元测试?

  1. 大牛(好的开源项目)
  2. 保住面子(不被测试同学天天追bug)
  3. 领导要求

如何写单元测试（Junit）

java框架中可以使用Junit做单元测试，非常简单。使用maven(gradle)构建java项目，会有专门写单元测试的目录。src -> test -> java

  1. 引入junit jar包。spring-boot已经集成了junit。可以直接使用spring-boot官方提供的test包
  1. 书写单元测试类

  1. 使用断言

上述代码中在测试中使用System.out.println进行输出结果，这种是不符合单元测试规范的，每次需要肉眼去看结果，我们可以使用断言进行判断结果是否符合预期。这样就可以全局跑单测，在版本迭代时全局运行后，没有错误就说明是正常的，可以发版。

  1. mock server

单测中调用a服务的方法，可能a中有调用b服务的方法，我们在单测中想只关注a服务的方法，这个时候可以对b服务的方法进行mock(创建一个虚拟的对象来方便进行测试)。
